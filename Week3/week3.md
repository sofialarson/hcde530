# Competency 2 — Code Literacy and Documentation (Week 3)

**Course definition:** *“Code Literacy and Documentation. Reading code well enough to explain what it does, change it when needed, and document it so someone else -- or future you -- can follow it. This icludes inine comments, docstrings, commit messages, and README content.”*

## Claim

For this week, I created a python script that intakes messy survey data on product satisifaction and analyzes it to output a cleaned data file that provides summary statistics on the response # by role, names of those who reported the highest satisifaction scores, and a list of products that were mentioned in the data. In this script, I added 1-3 comments per block of code, such as on Line 14, which says "Creates a list of products mentioned in the survey responses" to describe the overall chunk of code, providing a general overview for a user skimming the code. I then added a follow-up comment within that chunk, at line 22, stating "Extracts the product from the row and adds it to the set list of products that will be outputted." which highlights within that code how each individual instance of a product is added to a list.
